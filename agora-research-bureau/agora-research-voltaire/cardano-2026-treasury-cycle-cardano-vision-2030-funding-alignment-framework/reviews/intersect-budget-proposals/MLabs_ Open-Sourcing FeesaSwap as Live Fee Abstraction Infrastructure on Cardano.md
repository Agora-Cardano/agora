## AI Rationale

This proposal aligns most strongly with **Pillar 2: Adoption & Utility** because its core purpose is to reduce user friction in Cardano transactions by enabling users to pay fees with non-ADA tokens, while making the implementation open-source and easier for wallets, dApps, auditors, and builders to integrate.

The Cardano Vision 2030 framework defines Pillar 2 around widespread non-speculative utility, improved user experience, and developer experience, including “Invisible technology” and open-source incentives for SDKs, frameworks, and infrastructure. The FeesaSwap proposal directly targets that surface by making ADA fee requirements less visible to end users and by publishing reusable fee-abstraction infrastructure for builders. 

A plausible alternative would be **Infrastructure & Research**, because the proposal includes backend hardening, telemetry, monitoring, database reliability, and chain-access reliability. However, those are implementation means. The strategic purpose is not L1 protocol improvement, L2 integration, cross-chain interoperability, ZK infrastructure, post-quantum readiness, or client diversity. The funded work exists mainly to improve application usability, integration readiness, and transaction completion, which fits Pillar 2 more directly.

## Supporting Evidence

### Evidence 1

The proposal identifies the core barrier as the need for users to acquire ADA before transacting, which creates onboarding friction, exchange/KYC detours, and lost conversions. 

### Evidence 2

FeesaSwap is described as live mainnet fee-abstraction infrastructure that lets users pay transaction fees in non-ADA tokens while ADA and collateral are handled behind the scenes. 

### Evidence 3

The work package aims to open-source and harden FeesaSwap so wallets, dApps, auditors, infrastructure providers, and builders can inspect, integrate, reuse, and extend it. 

### Evidence 4

The proposal explicitly links its strongest alignment to Pillar 2’s “Invisible technology” focus, arguing that fee abstraction makes ADA fees less visible and smooths transaction flows. 

### Evidence 5

The expected value is framed around smoother onboarding and clearer integration paths for DeFi, payments, games, mints, bridges, RWAs, and other native-asset-first use cases.

## Classification Notes

### Primary Classification Justification

**Adoption & Utility** is the best primary classification because the proposal’s main strategic purpose is to make Cardano applications easier to use and easier to integrate by reducing ADA-first fee friction. This maps directly to Pillar 2’s focus on user experience, non-speculative utility, developer experience, and lower adoption friction.

**Infrastructure & Research** was not selected because the proposal does not primarily target protocol-level scalability, L1 consensus/ledger/networking improvements, L2 settlement architecture, cross-chain state proofs, ZK capabilities, post-quantum readiness, or full-node/light-client diversity. The technical hardening work supports the fee-abstraction product, but it does not define the proposal’s strategic purpose under Pillar 1.

### Additional Notes

No secondary pillar is assigned. The proposal has meaningful technical work, but those deliverables serve the Pillar 2 objective rather than constituting a separate, substantial Pillar 1 alignment under the Vision 2030 framework.
