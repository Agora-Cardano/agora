## AI Rationale

The proposal’s dominant strategic purpose is to advance Leios from an early public testnet toward a mainnet-ready release candidate. Its core work is consensus-layer engineering: enhancing Ouroboros Praos with Endorser Blocks and committee-based validation, increasing transaction throughput, conducting conformance testing against formal specifications, integrating into the primary node implementation, performing load and adversarial testing, and preparing hard-fork-enabling technical outputs.

This maps most directly to **Pillar 1: Infrastructure & Research**, especially the Vision 2030 focus on L1 protocol improvements, scalability, interoperability, security, and resilience. The proposal is not primarily an adoption program, ecosystem growth initiative, governance reform, or treasury sustainability mechanism. Those outcomes are presented mainly as downstream effects of improved protocol capacity.

Although the proposal references adoption, protocol revenue, SPO viability, and ecosystem readiness, these are not the proposal’s primary funded activities. The funded work is infrastructure-first: building, testing, validating, and preparing a core consensus upgrade.

## Supporting Evidence

### Evidence 1

The proposal states that it “funds the path from public testnet to a mainnet-ready release candidate,” delivering a “10–65x increase in transaction capacity.”

### Evidence 2

The proposal describes Leios as enhancing “Cardano's existing Ouroboros Praos consensus with Endorser Blocks and committee-based validation,” increasing throughput while preserving decentralization and stake pool viability.

### Evidence 3

The proposal’s first major objective is a “Release Candidate,” involving “substantial engineering,” “conformance testing against formal specifications,” “Leios block structure,” and “integration into the primary node implementation.”

### Evidence 4

The second objective, “High Confidence,” focuses on protocol validation through “parameter exploration,” “continuous load testing,” “adversarial testing,” “red-teaming exercises,” and an “updated threat model with validated mitigations.”

### Evidence 5

The third objective, “Hard-fork Enabling Leios,” includes stable client interfaces, technical documentation, support for close-by infrastructure projects, testnet hard-forks, mainnet parameter planning, contingency procedures, and governance artifact preparation.

## Classification Notes

### Primary Classification Justification

**Infrastructure & Research** was selected because the proposal’s core purpose is to develop and validate a major L1 consensus and scalability upgrade. Vision 2030 defines this pillar around keeping Cardano secure, fast, and interoperable, including L1 protocol improvements that raise throughput and improve finality. The Leios proposal fits this directly.

**Adoption & Utility** was not selected as the Primary Pillar because the proposal does not directly fund DeFi, RWA, payments, supply-chain adoption, user acquisition, enterprise adoption, or application-layer usage. It supports adoption indirectly by creating more capacity for future usage.

**Ecosystem Sustainability & Resilience** was not selected as a Secondary Pillar because the proposal’s sustainability arguments are mostly downstream: higher throughput may enable future transaction volume and protocol revenue, but the proposal itself acknowledges that revenue generation requires adoption. It does not directly implement treasury management, tokenomics reform, L2-to-L1 value retention, or SPO incentive programs.

**Community & Ecosystem Growth** was not selected because SPO/developer workshops and ecosystem coordination are supporting activities for hard-fork readiness, not the main strategic purpose.

**Governance** was not selected because governance artifact preparation is procedural support for a protocol upgrade, not a direct governance-system improvement.

### Manual Review Reason

Not applicable.

### Additional Notes

The proposal contains references to multiple Vision 2030 outcomes, but they are not equally weighted. The funded deliverables are overwhelmingly technical and infrastructure-focused, so no manual review is required under the stated criteria.
