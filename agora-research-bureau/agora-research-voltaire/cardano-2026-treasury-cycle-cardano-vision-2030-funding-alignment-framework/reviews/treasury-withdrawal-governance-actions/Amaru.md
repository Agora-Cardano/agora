## AI Rationale

Amaru’s primary strategic purpose is to build and mature an alternative, interoperable, block-producing Cardano node. That maps directly to **Pillar 1: Infrastructure & Research Excellence**, especially the Vision 2030 focus on keeping Cardano “secure, fast, and interoperable,” improving protocol infrastructure, and supporting client diversity. The proposal is centered on relay capability, block production, hard fork integration, protocol experimentation, conformance testing, security audits, network-level testing, and middleware integrations.

The strongest alignment is not adoption, governance, or treasury sustainability. Those may appear around the edges, but the funded work itself is core protocol/node infrastructure. The Vision 2030 framework explicitly places additional full-node implementations and conformance testing under Infrastructure & Research, making this classification highly direct.

## Supporting Evidence

### Evidence 1

The proposal states that “Amaru is an open-source project building a new, fully interoperable block-producing node that improves the overall accessibility and robustness of the Cardano network without compromising its safety and security.”

### Evidence 2

The proposal identifies key 2026 deliveries including “a relay-capable node in Q1 2026,” “a block-producing node by the end of Q2 2026,” and integration with upcoming protocol changes such as an intra-era hard fork.

### Evidence 3

The proposal includes protocol-level experimentation and prototyping for “planned improvements to the protocol,” including Leios, Peras, Phalanx, Decentralized Messaging Queue, StarStream, and Midgard.

### Evidence 4

The proposal includes a dedicated scope for “Network Compliance & Cardano-level Testing,” with the objective of creating “a robust Cardano network level testing environment” to find vulnerabilities and make the Cardano network more resilient.

### Evidence 5

The proposal states that Amaru represents “a major step towards node diversity by developing a modular, high-performance, and interoperable block-producing node for Cardano.”

## Classification Notes

### Primary Classification Justification

**Infrastructure & Research** was selected because the proposal’s core deliverables are a Cardano node implementation, network conformance testing, protocol compatibility, performance optimization, security auditing, and integrations with technical middleware. These match Vision 2030 Pillar 1, especially the areas covering L1 protocol improvements, security and resilience, client diversity, and interoperability.

**Adoption & Utility** was not selected as the Primary Pillar because the proposal does not primarily target end-user adoption, real-world verticals, DeFi, RWA, payments, consumer UX, or enterprise adoption. Some developer and middleware benefits exist, but they are instrumental to node infrastructure rather than the main strategic purpose.

**Governance** was not selected because governance references are mostly administrative or compliance-related. The proposal does not primarily develop governance participation tools, voting mechanisms, DRep incentives, treasury seasons, or turnout-aware governance systems.

**Community & Ecosystem** was not selected because workshops, contributor attraction, SPO engagement, and communication are supporting activities. They help execute and disseminate the infrastructure work but do not define the proposal’s core purpose.

**Sustainability & Resilience** was not selected as a Secondary Pillar because, although the proposal frequently references resilience, node diversity, and long-term sustainability, those concepts are already directly covered under Pillar 1’s security, resilience, and client diversity areas. The proposal does not substantially target Pillar 5’s treasury management, tokenomics, L2-to-L1 value retention, or SPO incentive model as its own strategic workstream.

### Manual Review Reason

Not applicable.

### Additional Notes

Secondary Pillar is set to **None** because all substantial workstreams remain within the same strategic cluster: core Cardano infrastructure, node diversity, network testing, protocol compatibility, and resilience. Any adoption, community, or governance effects are indirect or operationally supportive.
